# 06-output-specification.md

# Output Specification

## Required Folder Structure

Create the following files:

- `source_notes.md`
- `chapter-index.md`
- `compassion-benchmark-funder-prospectus.md`
- `graphics-plan.md`
- `layout-notes.md`
- `funder-summary.md`
- `claims-to-verify.md`
- `source-map.md` if useful

## Main Manuscript

File name:

`compassion-benchmark-funder-prospectus.md`

Required title:

# Compassion Benchmark

## Building the Global Standard for Measuring Institutional Compassion

### A Strategic Vision, Research Prospectus, and Funding Opportunity

Target:

- 25,000 to 35,000 words
- 60 to 70 designed pages
- publication-ready Markdown
- clear hierarchy
- no placeholder text
- no unresolved TODOs in the manuscript

## Source Notes

File name:

`source_notes.md`

Include:

- summary of each source document
- useful facts
- methodology details
- governance details
- funding logic
- public benefit claims
- risks
- unresolved questions
- source material that should not be used publicly without revision

## Chapter Index

File name:

`chapter-index.md`

Include a table with:

- chapter number
- chapter title
- purpose
- target word count
- core takeaway
- key source material
- recommended visual
- engagement point
- claims to verify

## Graphics Plan

File name:

`graphics-plan.md`

Include:

- cover image recommendation
- section divider concepts
- eight-dimension icon system
- evidence hierarchy graphic
- theory of change graphic
- platform workflow
- roadmap timeline
- funding package graphic
- impact framework
- suggested pull quotes
- suggested sidebar concepts
- recommended chart style

## Layout Notes

File name:

`layout-notes.md`

Include:

- page rhythm guidance
- typography recommendations
- chapter opening treatment
- callout styles
- pull quote treatment
- tables and charts treatment
- image use principles
- white space guidance
- PDF production notes

## Funder Summary

File name:

`funder-summary.md`

Create a 3 to 5 page condensed version suitable for:

- grant attachments
- introductory funder emails
- one-on-one funder conversations
- board review

It should include:

- problem
- solution
- public benefit
- progress to date
- funding priorities
- governance
- sustainability
- impact expectations

## Claims to Verify

File name:

`claims-to-verify.md`

Include:

- claim
- location in manuscript
- why verification is needed
- suggested source type
- recommended wording if verification is not yet available

## Markdown Requirements

Use clean Markdown.

Use heading levels consistently.

Do not use HTML unless necessary.

Do not use decorative Unicode lines or excessive formatting.

Tables are acceptable where they improve readability.

## Completion Summary

At the end of the Claude Code run, provide:

1. Brief completion summary
2. File list
3. Remaining assumptions
4. Claims needing verification
5. Suggested next steps for layout and PDF production
