# Writing Workflow

## Overview

Do not attempt to create the full publication in one unreviewed pass.

Work in controlled stages:

1. Source ingestion
2. Source notes
3. Chapter index
4. Draft chapters
5. Rewrite chapters
6. Human voice pass
7. Funder readiness pass
8. Evidence and claims pass
9. Layout pass
10. Final assembly

Every output file should be usable.

## Stage 1: Source Ingestion

Read all available source materials in the project folder.

Create `source_notes.md`.

Capture:

- mission and vision statements
- governance decisions
- legal structure options
- nonprofit conversion considerations
- methodology details
- scoring formulas
- evidence hierarchy
- assessment protocols
- platform capabilities
- product and service lines
- existing revenue assumptions
- funding logic
- public benefit framing
- risks and unresolved claims

Do not copy the source text mechanically.

## Stage 2: Chapter Index

Create `chapter-index.md`.

For each chapter include:

- chapter title
- purpose
- core reader takeaway
- target word count
- key source material
- recommended visual
- likely evidence needs
- expected engagement point

Use the chapter index as the blueprint for the manuscript.

## Stage 3: Chapter Drafting

Draft one chapter at a time.

Each chapter must contain:

- a strong human opening
- a clear argument
- concrete institutional examples
- connection to Compassion Benchmark
- funder relevance where appropriate
- recommended graphic or layout note
- no filler

Use headings sparingly.

Let paragraphs do the work.

## Stage 4: Developmental Rewrite

After drafting each chapter, review:

- Does the chapter need to exist?
- Does it advance the central argument?
- Does it repeat earlier material?
- Does it prepare the next chapter?
- Is there a real idea on the page?

Rewrite before moving on.

## Stage 5: Human Voice Pass

Remove AI-sounding prose.

Look for:

- repeated sentence length
- repetitive paragraph structure
- over-explaining
- fake profundity
- vague abstractions
- generic nonprofit language
- stiff transitions
- empty summaries

Rewrite until the chapter sounds like a human author with judgment and experience.

## Stage 6: Funder Readiness Pass

For every chapter ask:

- Why would a funder care?
- What public benefit is created?
- Why is philanthropy needed?
- What work becomes possible if funded?
- How is independence protected?
- What is the long-term value?

Do not add grant language everywhere. Make the relevance clear through the argument.

## Stage 7: Evidence and Claims Pass

Create `claims-to-verify.md`.

Flag claims that require:

- citation
- updated market data
- legal review
- nonprofit tax review
- defamation review
- methodology validation
- external evidence
- softer wording

Do not allow unsupported market size claims or legal claims to remain unflagged.

## Stage 8: Layout Pass

Create `layout-notes.md` and `graphics-plan.md`.

Every chapter should include:

- recommended visual
- pull quote candidate
- sidebar or callout suggestion
- page design note
- data visualization candidate if applicable

## Stage 9: Final Assembly

Assemble the full manuscript as `compassion-benchmark-funder-prospectus.md`.

Ensure consistent:

- title hierarchy
- chapter structure
- voice
- public brand language
- terminology
- spelling
- no em dashes
- no legacy brand drift

## Stage 10: Final Review

Run quality gates from `07-quality-gates.md`.

Do not mark the manuscript complete until all gates pass or unresolved items are documented.
