# Whole-Site Review — Consolidated Master Plan (2026-06-12)

Consolidated from four lenses: UX (`SITE_REVIEW_UX_*`), Knowledge/content (`SITE_REVIEW_KNOWLEDGE_*`), Architecture/code (`SITE_REVIEW_ARCHITECTURE_*`), Graphics (`SITE_REVIEW_GRAPHICS_*`). Goal: **improve AND simplify** the experience while keeping the critical information + research prominent, and **use graphics well**.

Scoring: **Priority = Impact + Strategic Alignment + Learning Value + Confidence − Effort − Risk**.

---

## The convergent finding (all four lenses agree)
The site got **deep faster than it got clear.** The recent entity-page waves added real analytical value but also **density and redundancy**, while the **highest-traffic surfaces (the 7 index pages) were left under-built** — bare acronym tables that fail the 5-second test. The fix is the same fix four times: **define the vocabulary once, show it visually, and simplify by removing duplication — not by removing substance.**

Five threads recur across the lenses:
1. **The entity page is now over-dense / redundant** *(UX #1–4, Knowledge #7)* — same facts said 3–4 ways (rank restated 3×; best/worst badges + radar + "if you remember one thing" all encode the dimension shape).
2. **The 7 index pages (highest traffic) fail the 5-second test** *(UX #8, Knowledge #1, Graphics I1)* — 8 undecoded acronym columns, and a plain HTML band table where the shipped `BandDistributionBar` should be.
3. **Vocabulary taught nowhere / inconsistently** *(Knowledge #1–3, cross-cutting)* — bands phrased 3 different ways; "integration premium" introduced as jargon; no "how to read a score" primer.
4. **Internal/monetization copy leaking onto public pages** *(UX #9/#11, Knowledge #4/#9)* — `/indexes` buyer-path + revenue tables, methodology `ACB-HAB-001` doc-IDs.
5. **Code carries ~1,500 churning generated files + ~2,200 dead lines + duplicated mappings** *(Architecture #1–3)* — low-risk cleanup that also ends the per-wave `public/data` churn I keep discarding.

## Single highest-leverage move

Highest traffic, fails the 5-second test today, the data + the primitive already exist, one change fixes 7 pages, near-zero risk. ## The honest note on the entity page
Three lenses independently flagged that the entity page (which we just spent Waves 1–3 enriching) is **now too dense and repeats itself**. S1.3 is the corrective: keep every distinct piece of *information*, remove the duplicate *encodings* (the radar already shows strongest/weakest, so the badge pair + the "if you remember one thing" box are redundant). Simplify by dedup, not by deletion of substance.

## Independence / integrity
All moves PASS. Graphics are own-data SVG (CC-BY, no third-party imagery). Removing the internal buyer-path/monetization tables from public index pages *strengthens* the independence posture. No content is fabricated; simplification is dedup + reuse.
